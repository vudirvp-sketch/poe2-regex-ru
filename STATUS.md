### Детерминированная стратегия регексов (8 принципов)

| # | Принцип | Описание |
|---|---------|----------|
| 1 | **One Mod = One Quoted Group** | Каждый выбранный мод → одна quoted group с suffix + `.*` bridge + number pattern |
| 2 | **Multi-Mod = AND Across Blocks** | N модов → N quoted groups через пробел: `"mod1" "mod2" "mod3"` |
| 3 | **NO Multi-Word `\|`** | `\|` между многословными альтернативами сломан на всех уровнях (Tests 9-17) |
| 4 | **`.*` Bridging Within Single Block** | `"prefix.*suffix"` мостит число и слова в одном блоке |
| 5 | **Suffix Uniqueness** | Найти кратчайший suffix, уникальный для мода в категории (≥3 значимых символов на слово) |
| 6 | **Shared Suffix → Differentiate by Number** | Если suffix shared → number enumeration: `"(1[0-5])%.*suffix"` |
| 7 | **Cross-Block FP Risk** | `"X" "Y"` (AND) может-match разные блоки → FP. Использовать `.*` bridge: `"X.*Y"` |
| 8 | **Same-Family OR** | OR между same-family модами невозможен в текущем диалекте → UI redesign или AND fallback |

| `\|` между многословными (верхний уровень) | ❌ | Tests 9, 10, 11 |

### Три бага (подтверждены)

**Bug A: `|` не работает с многословными паттернами (CRITICAL — ПОДТВЕРЖДЁН)**

PoE2 токенизирует поисковую строку по пробелам. `|` OR-ит только одиночные слова. Многословные альтернативы через `|` не работают ни в каком контексте.

| Категория | Opt-записей с многословным `\|` | Всего opt-записей | % сломанных |
|-----------|-------------------------------|-------------------|-------------|
| Самоцветы | 106 | 113 | 94% |
| Амулеты | 53 | 114 | 46% |
| Кольца | 44 | 93 | 47% |
| Пояса | 41 | 83 | 49% |
| Плиты | 26 | 35 | 74% |
| Путевые | 13 | 43 | 30% |

**Bug B: AND-in-OR = вложенные кавычки (CRITICAL)**

`compiler.ts` compileInner(AND) оборачивает children в `"..."`, даже внутри OR. PoE2 не поддерживает вложенные кавычки.

**Bug C: regexExclude — неполные словоформы (MEDIUM)**

`самострелами` ≠ `самострела`. Нужно использовать усечённые основы: `самострел`, `посох`, `копь`.

### Путь B: Полный редизайн (ПОДТВЕРЖДЁН, стратегия определена)

Путь A (исправить только компилятор) исключён — `()` + `|` + многословные НЕ работают.

#### Стратегия редизайна (детерминированная, 8 принципов выше)

Ключевая идея: **заменить многословные `|`-альтернативы на комбинации `.*`, `(?!…)`, AND и number-enumeration**.

| Шаг | Содержание | Затронуто | Сложность |
|-----|-----------|-----------|-----------|
| **B0** | **Критический тест в игре:** работает ли `"A B"\|"C D"`? | — | Тест в игре (3 варианта готовы) |
| B1 | ETL: переработать compute-optimizations.ts — заменить многословные `\|` на per-mod quoted groups | `compute-optimizations.ts` | Высокая |
| B2 | Компилятор: AND-in-OR генерировать через `()` вместо вложенных `"..."` | `compiler.ts` | Средняя |
| B3 | ETL: regexExclude → усечённые основы + `(?!…)` per-block | `compute-regex-strategies.ts` | Средняя |
| B4 | Runtime: обновить `buildLiteralNode` / `applyOptimizationTable` | `optimization-strategies.ts`, `useCategoryPage.ts` | Средняя |
| B5 | In-game верификация: самоцветы (4 тестовых предмета готовы) | — | — |
| B6 | Распространение: амулеты, кольца, пояса, плиты, путевые | — | — |

**⚠️ Шаг B0 — приоритетный.** Если `"A B"|"C D"` НЕ работает, то OR между same-family модами невозможен → нужен UI redesign (каждый OR-child = отдельный AND-фильтр, взаимоисключающий выбор).
