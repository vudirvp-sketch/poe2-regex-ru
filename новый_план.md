## Известные оставшиеся проблемы

### 🔴 P0 — Критично

#### 1. Push фиксов в GitHub
Локальные TS-фиксы НЕ запушены в main → GitHub Actions продолжает падать.

#### 2. Проверка regex-строк в игре
Тестировать КАЖДЫЙ регис ОТДЕЛЬНО (не AND!). См. IN_GAME_TESTS.md тесты 1-22.
- Если `\d` не работает в игре — заменить на `[0-9]` в `number-regex.ts`

#### 3. jewelType — все "shared"
Type A парсер (jewels) не извлекает `modCode` → `buildJewelTypeMap()` не может смэтчить.
Все 193 jewel-токена получают `jewelType: "shared"`. Runtime-классификатор (~84%) работает, но ETL-классификация не функционирует.
**Фикс**: нужно извлечь modCode из Type A HTML-таблиц или сопоставить по rawText.

### 🟡 P1 — Важно

#### 4. Desecrated regex quality
Дуал-стат моды с regex типа `"и, (4—8)% увеличение урона т"` — содержит число и запятую. Улучшить Strategy 1c в `compute-regex.ts`.

#### 5. Полная проверка Очернённых по всем категориям
Убедиться что normal/desecrated/corrupted моды корректно разделены во ВСЕХ категориях.

### 🟢 P2 — Желательно

#### 6. Виртуализация списков (belt 298, ring 366, amulet 427)
#### 7. UX: авто-копирование, клавиатурные шорткаты, health bar анимация

## Архитектурные инварианты (НЕ НАРУШАТЬ)

```
I1. Лимит символов = 250 (str.length, НЕ TextEncoder)
I2. Core Layer (src/core/) — ZERO зависимостей
I3. public/generated/ — READ-ONLY, создаётся ТОЛЬКО ETL-скриптами
I4. Никаких хардкод-строк модов в UI/Engine. Только internal_id + i18n
I5. pnpm — единственный пакетный менеджер
I6. Правило 3: Не создавать абстракцию, пока логика не написана 3 раза
I7. Locale = 'ru' сейчас. Типы поддерживают расширение
I8. PoE2 regex диалект: . (wildcard), | (OR), ! (NOT), "" (группировка), [] (char class)
```
