### P4: Suffix anchoring investigation
- Проверить в игре: работает ли `"(2[7-9]|30)%.*откладывания наград"` — `%` после числа
- Для `%`-модов: `%` стоит после актуального ролла, но НЕ после числа из range notation
- Осторожно: `50(27-50)%...` → актуальный ролл `50` сопровождается `(`, не `%` → suffix anchoring создаёт FN!
- Может подойти только для модов БЕЗ range notation на предмете — сложно использовать

### Баг-профиль для будущего аудита
1. **Range notation FP для аксессуаров** — `+`-префиксные моды не получают `^` anchor. FP от range notation может происходить на предметах с `+26(27-50)%`. `+` перед числом частично предотвращает FP (паттерн `([2-9][0-9])` может совпасть с "27" из range notation), но не полностью. Suffix anchoring (`%`) может помочь, но создаёт FN на предметах с range notation у актуального ролла.
2. **Edge cases с OR-suffix RANGE** — при разных (min,max) но одинаковых суффиксах, объединение может быть некорректным
3. **VendorPage numeric-only без чекбокса** — свойство с numericInput, но без чекбокса (selectedIds), может не попасть в regex
