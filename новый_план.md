## Выполнено в итерации 21

### ✅ ETL re-run + критические баг-фиксы

1. **Fractional ranges** — `normalize.ts` теперь парсит дробные диапазоны `(2.1—3)` через `parseFloat`
2. **Suffix lengthening** — `compute-regex.ts`: при неуникальном суффиксе (напр. "к атакам") алгоритм пробует extended suffix (текст после первого `##`), затем fallback на substring search по rawText. Результат: `"огня к атакам"`, `"холода к атакам"`, `"молнии к атакам"` вместо конфликтующего `"к атакам"`
3. **Desecrated dual-stat mod <br> split** — `normalize.ts`: когда несколько `<br>`-сегментов содержат `<span class="mod-value">` с ndash, они объединяются через `", "` (как в игре). Результат: `jewel-desecrated` вырос с 21→32 токенов (дуал-стат моды теперь как одна строка)
4. **Origin classification fix** — `mod-classifier.ts`: режим `'origin'` теперь использует `splitGroupByOrigin()` вместо первого-non-normal происхождения. Нормальные моды больше НЕ показываются в секции "Очернённые"
5. **вамиИстощения typo fix** — добавлен в `i18n-overrides.json`: `вами Истощения` (пробел восстановлен)
6. **FilterChip prefix indicator** — ⚓ иконка показывается когда у выбранного чипа есть regexPrefix
7. **TS type fixes** — `regexPrefix` добавлен во все тестовые хелперы

### ⚠️ Известные оставшиеся проблемы

- **Desecrated regex quality**: некоторые дуал-стат моды получают regex из fallback substring search (напр. `"и, (4—8)% увеличение урона т"` — содержит число и запятую). Это не идеально, но уникально и функционально.
- **Regex в игре**: пользовательские тесты регексов не прошли из-за AND vs OR путаницы (каждая пара кавычек = AND условие). Нужно тестировать каждое условие отдельно.

## Приоритетные задачи (следующая итерация)

### 🔴 P0 — Критично

#### 1. Проверка regex-строк в игре
Тестировать КАЖДЫЙ регис ОТДЕЛЬНО, не все вместе:
- `"(1[5-9]|[2-9].|\d..).*повышение брони"` — для ≥15% брони
- `"(1[1-9]|[2-9].|\d..).*хаосом"` — для ≥11% урона хаосом
- Если `\d` не работает в игре — заменить на `[0-9]`

#### 2. Полная проверка Очернённых по всем категориям
Убедиться что normal/desecrated/corrupted моды корректно разделены во ВСЕХ категориях, не только самоцветах.

### 🟡 P1 — Важно

#### 3. Двухчисловые моды — фильтр по первому числу
"От ## до ## физического урона" — сейчас фильтр по второму числу. Нужно фильтровать по первому.

#### 4. PoE2 `\d` поддержка
Проверить поддерживает ли PoE2 `\d` в поиске. Если нет — заменить все `\d` на `[0-9]` в `number-regex.ts`.

#### 5. Desecrated regex quality improvement
Дуал-стат моды с regex типа `"и, (4—8)% увеличение урона т"` — улучшить алгоритм для более чистых регексов.

### 🟢 P2 — Желательно

#### 6. Виртуализация списков (belt 298, ring 366, amulet 427)
