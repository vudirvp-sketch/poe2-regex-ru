# PoE2 Regex RU — План доработок

> **Версия:** 8.0 | **Дата:** 2026-06-07

---

## Текущий статус проекта

- **8 страниц категорий** + VendorPage
- **437 тестов** — все проходят
- **Блок-модель матчинга реализована** — `matchPoE2RegexItem()` + `getItemSearchBlocks()`
- **Prefix anchoring упрощён** — только для dual-number модов (cross-mod FP не существует)
- **In-game тесты Phase 7 завершены** — 13 блоков, ~30 тестов, результаты интегрированы
- **`.*` НЕ пересекает границы блоков** — фундаментальное открытие
- **Описание НЕ индексируется** — `description` field добавлен
- **`?` НЕ работает** в PoE2 regex

1. **Re-run ETL** — `pnpm etl` чтобы перегенерировать JSON с упрощённым prefix anchoring

3. **Интеграция `matchPoE2RegexItem` в Oracle** — Oracle должен учитывать блок-модель
4. **Удалить deprecated `getItemSearchText()`** — заменить все вызовы на `getItemSearchBlocks()`
5. **jewel-corrupted** — avg regex len = 7.4, добавить в STRICT_CATEGORIES

### 🟢 P2 — Желательно

6. **Фазы 3-8 оптимизатора** — DP на Trie, диалектные оптимизации, итеративный цикл
7. **Виртуализация списков** (belt 298, ring 366, amulet 427)
8. **Multi-line mod handling** — Разрушительный: каждая под-строка = отдельный блок в ETL
