## Текущий статус проекта

Проект **poe2-regex-ru** — генератор regex-строк для русского клиента Path of Exile 2. Все основные функции реализованы и работают:

- **8 страниц категорий**: Путевые камни, Плитки, Реликвии, Самоцветы, Торговец, Пояса, Кольца, Амулеты
- **109 тестов** — все проходят
- **1,584 токена** в 10 категориях данных (ETL из poe2db.tw)
- **Оптимизатор** с дедупликацией + optimization table
- **Профили** (сохранение/загрузка фильтров в localStorage)
- **URL-шеринг** (lz-string сжатие + extraState для категории-специфичных переключателей)
- **Health Bar** (визуальный индикатор 0/250 символов)
- **Адаптивная вёрстка** (hamburger-меню, slide-in sidebar)
- **SEO мета-теги** (Open Graph, Twitter Card, canonical URL)
- **CI/CD** (GitHub Actions: ETL → Build → Deploy на GitHub Pages)

### 🟡 P1 — Важно (ухудшает UX)

#### 3. Обновление данных ETL
**Проблема:** Данные в `public/generated/` устарели — ETL последний раз запускался 2025-06-05. poe2db.tw мог обновить моды.

**Решение:** Запустить `pnpm etl` локально (нужен интернет) и закоммитить обновлённые JSON.

**Риск:** poe2db.tw может изменить HTML-структуру, что сломает парсеры. Нужно проверить вывод ETL на предмет ошибок.

#### 4. ~50 токенов с английским rawText
**Проблема:** poe2db.tw не имеет русских переводов для ~50 модов. В UI они отображаются на английском.

**Решение:** Варианты:
- (A) Дождаться обновления poe2db.tw
- (B) Добавить ручные переводы в отдельный override-файл, применяемый после ETL
- (C) Пометить в UI как "нет русского перевода"

**Оценка:** 2-3 часа на вариант B.

#### 5. Полная поддержка min+max RANGE
**Проблема:** Когда указаны и min, и max в RANGE, используется только min. Полный диапазон min ≤ x ≤ max не генерируется.

**Решение:** Расширить `compiler.ts` для генерации пересечения min-regex и max-regex. Сложная задача из-за диалекта PoE2 regex.

**Оценка:** 4-6 часов.

## Архитектурные инварианты (НЕ НАРУШАТЬ)

```
I1. Лимит символов = 250 (str.length, НЕ TextEncoder)
I2. Core Layer (src/core/) — ZERO зависимостей. Никаких React, DOM, Zustand.
I3. public/generated/ — READ-ONLY. Создаётся ТОЛЬКО ETL-скриптами.
I4. Никаких хардкод-строк модов в UI/Engine. Только internal_id + i18n.
I5. pnpm — единственный пакетный менеджер.
I6. Правило 3: Не создавать абстракцию, пока логика не написана 3 раза.
I7. Locale = 'ru' сейчас. Типы поддерживают расширение.
I8. PoE2 regex диалект: . (wildcard), | (OR), ! (NOT), "" (группировка), [] (char class), ^/$ (anchors)
```
