## 6. Key Architecture Points

### PoE2 Regex Dialect (NOT standard PCRE)

| Syntax | Meaning | Verified |
|--------|---------|----------|
| `substring` | Simple substring match | ✅ |
| `\|` | OR (alternation) | ✅ |
| `!` | NOT (negation) | ✅ |
| `""` | Phrase grouping + AND separator | ✅ |
| `.` | Any single character | ✅ |
| `.*` | Any sequence WITHIN a single block | ✅ |
| `[]` | Character class | ✅ |
| `^` | Start-of-block anchor | ✅ |
| `()` | Grouping | ✅ |
| `\d` | Digit shorthand | ✅ |
| `{N,}` | Quantifier "N or more" | ✅ |
| `\d{3,}` | ≥100 compact | ✅ |
| `\d{2,}` | ≥10 compact (replaces broken `[0-9][0-9][0-9]?`) | ✅ |
| `!` + `[]` / `\d` | Negation with regex groups | ✅ |

**NOT supported:** `?` (optional), `.*` across blocks, negative lookahead, non-greedy quantifiers, backreferences. `$` end anchor is unreliable — do not use.

**Critical rules:**
1. `!` must be INSIDE quotes when combined with `|`: `"!A|B"` works, `!"A|B"` does NOT
2. `.*` does NOT cross block boundaries — each mod/implicit/property is a separate block
3. `.*` is directional — forward only within a block
4. AND via space between quoted groups is order-independent and works ACROSS blocks
5. `!X` is item-wide — excludes entire item if X appears in ANY block
6. Case insensitive — verified with Cyrillic

### Block-Based Matching Model

Each piece of item text is an independent searchable block. `.*` works ONLY within a single block. AND (`"X" "Y"`) works ACROSS blocks.

### 4-Level FP Prevention

| Level | Method | When | FP prevented |
|-------|--------|------|-------------|
| 1 | `^` (anchorStart) | Template starts with `##` | Range notation at non-position-0 |
| 2 | `%` suffix anchor (anchorEnd) | Template has `##%` AND anchorStart=false | Numbers not followed by `%` |
| 3 | Enumeration (compact decade) | Range ≤ 50 values | Secondary numbers not matching enumerated values |
| 4 | Threshold ≥N% | `([X-Y]\d|\d{3,})%.*suffix` | Range notation — verified NO FP |

### Compiler: 3 Compilation Strategies

- **Enumerated** (≤50 values): single quoted group with compact decade grouping → `"(2[7-9]|30).*suffix"`
- **Threshold** (threshold=true): single quoted group with ≥min only → `"(2[7-9]|[3-9]\d|\d{3,})%.*suffix"` — no FP from range notation, saves 30-50 chars vs enumeration for wide ranges
- **AND Fallback** (>50 values, no threshold): two AND-joined groups → `"≥min.*suffix" "≤max.*suffix"`

### Optimizer: 3 Phases

| Phase | Purpose | What it does |
|-------|---------|-------------|
| 1 | Deduplication | Collapse identical regex in OR groups into single LITERAL |
| 2 | Optimization table | Replace groups of LITERALs with shorter shared substrings |
| 3 | Truncated tails | Shorten suffixes using verified safe list |

**Phase 3 safe list:** `эффективн`, `бездн`, `путев`, `глубин`
**Phase 3 blacklist:** `редкост` (FP on rarity label "редкий"), `редк`, `провал`

### Number Regex: `\d{3,}` and `\d{2,}`

- `\d{3,}` replaces `[0-9][0-9][0-9]` for "any 3+ digit number" — saves 9 chars per occurrence
- `\d{2,}` replaces `[0-9][0-9][0-9]?` for "any 2+ digit number" — fixes `?` not supported bug
- `\d{4,}` added for ≥200, ≥300 etc. to handle 4+ digit numbers correctly
- `[0-9]` still used for single-digit character classes and within decade segments (e.g., `3[0-9]`)

### Positive + Negative Mods (Want + Don't-Want)

Pattern: `"want1|want2" "!dontwant1|dontwant2"`

Architecture: `excludeMode=false` → positive matches, `excludeMode=true` → EXCLUDE(OR(...)). Combined in `buildAstFromSelections()`.

### 250-Char Budget

Budget estimation in `limits.ts`. Optimization layers: family grouping, DP factorization, suffix shortening, yofication, optimization table, truncated tails.

## 7. ETL Pipeline Overview

```
poe2db.tw/ru → fetch → normalize → filter implicits → compute regex
→ compute optimizations → generate JSON → i18n overrides → FP repair
→ patch opt entries → iterative optimizer → public/generated/*.json
```

Full details in `docs/ETL_GUIDE.md`.

**i18n overrides:** `scripts/etl/i18n-overrides.json` — manually verified Russian translations for tokens that poe2db.tw shows in English only (~56 overrides).

**Source hash:** Each generated JSON contains `sourceHash` — SHA-256 of source HTML. `pnpm etl:check-stale` compares against cache to detect data changes.

## 8. Data Flow (Runtime)

```
public/generated/*.json
    → loader.ts (fetch + cache)
    → UI: user selects filters (want + don't-want via exclude mode)
    → useCategoryPage hook (group tokens into FamilyGroups)
    → ast.ts (build AST from selections — AND/OR + EXCLUDE)
    → optimizer.ts (Phase 1: dedup, Phase 2: optimization table, Phase 3: truncated tails)
    → compiler.ts (compile AST → regex string)
    → Regex displayed in RegexOutput → copied → pasted in PoE2 search
```

## 11. Frequent Pitfalls

1. **`!` must be INSIDE quotes with `|`:** `"!A|B"` works, `!"A|B"` does NOT
2. **`.*` does NOT cross block boundaries** — use AND (`"X" "Y"`) for cross-block search
3. **`$` end anchor is unreliable** — do not use
4. **`?` optional quantifier NOT supported** — use `\d{2,}` instead of `[0-9][0-9]?`
5. **Implicit-set bonuses are NOT searchable** — filtered out in ETL Step 2b
6. **Core must stay dependency-free** — never add npm imports to `src/core/`
7. **Generated JSON is read-only** — never edit `public/generated/` by hand
8. **Types in `types.ts` ONLY** — no duplicate interface definitions elsewhere
9. **Vendor properties in `vendor-properties.ts` ONLY** — canonical source
10. **`round10` always disabled for enumerated ranges** — enumeration is inherently precise
11. **`reversed=true` for implicit tokens** — implicit blocks have number AFTER text, regex pattern is `"suffix.*(number)%"`
12. **Colon anchor (`: `)** — for non-% reversed mods ending with `: ##`, prevents FP from range notation
13. **Word truncation works at END only** — `"к си"` matches `"к силе"`, mid-word does NOT work. Min 3 significant chars.
14. **Minimum regex length** — optimizer enforces min 5/7/10 chars by category; shorter gets `regexPrefixContext`
15. **DIALECT_PAIRS** — only genuine phonological alternates (е/ё, ю/я, а/я, ы/е, и/е). Never add о→в or similar non-alternating pairs
16. **VendorPage uses `useCategoryPage` with `customData`** — `buildVendorCategoryData()` converts VENDOR_PROPERTIES into CategoryData format. GROUP_COLORS derived from `tags: [group:${group}]`
17. **Vendor numeric properties use reversed pattern** — `"suffix.*number"` not `"number.*suffix"` because game property lines have text before number
18. **Item rarity label IS indexed** — "редк" from "редкий" matches ALL rare items. NEVER use `редкост` as truncated anchor — FP guaranteed. Blacklisted in optimizer Phase 3
19. **`{N,}` quantifier works** — `\d{3,}`, `\d{2,}`, `\d{4,}` all valid in PoE2 regex. Replaces `[0-9][0-9][0-9]` and the broken `[0-9][0-9][0-9]?`
20. **Threshold mode compiles RANGE(min,max) as ≥min only** — single quoted group, no FP from range notation, but drops max constraint. Use when budget is tight or range is wide
21. **Truncated tail optimizer runs automatically** — Phase 3 in `optimize()`. Only safe-listed truncations are applied. Blacklist prevents FP truncations like `редкост`
22. **`?` is NOT supported in PoE2 regex** — never use `?`, `{N,M}`, or `{N}?`. Use `{N,}` for "N or more" and enumeration for exact ranges
