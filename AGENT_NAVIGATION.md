## 6. PoE2 Regex Dialect (NOT standard PCRE)

| Syntax | Meaning | Verified |
|--------|---------|----------|
| `substring` | Simple substring match | ‚úÖ |
| `\|` | OR (alternation) | ‚úÖ |
| `!` | NOT (negation) | ‚úÖ |
| `""` | Phrase grouping + AND separator | ‚úÖ |
| `.` | Any single character | ‚úÖ |
| `.*` | Any sequence WITHIN a single block | ‚úÖ |
| `[]` | Character class | ‚úÖ |
| `^` | Start-of-block anchor | ‚úÖ |
| `()` | Grouping | ‚úÖ |
| `\d` | Digit shorthand | ‚úÖ |
| `{N,}` | Quantifier "N or more" | ‚úÖ |

**NOT supported:** `?` (optional), `.*` across blocks, negative lookahead, non-greedy quantifiers, backreferences. `$` end anchor is unreliable ‚ÄĒ do not use.

**Critical rules:**
1. `!` must be INSIDE quotes when combined with `|`: `"!A|B"` works, `!"A|B"` does NOT
2. `.*` does NOT cross block boundaries ‚ÄĒ use AND (`"X" "Y"`) for cross-block search
3. `.*` is directional ‚ÄĒ forward only within a block
4. AND via space between quoted groups is order-independent and works ACROSS blocks
5. `!X` is item-wide ‚ÄĒ excludes entire item if X appears in ANY block
6. Case insensitive ‚ÄĒ verified with Cyrillic
7. **Substring search** ‚ÄĒ PoE2 is substring match. Truncated words always work if the prefix is unique in context

## 7. 4-Level FP Prevention

| Level | Method | When | FP prevented |
|-------|--------|------|-------------|
| 1 | `^` (anchorStart) | Template starts with `##` or `[+-]##` | Range notation at non-position-0 |
| 2 | `\+` / `-` (signPrefix) | Template has `+##` or `-##` before number | Range notation numbers never have +/- sign |
| 3 | `%` suffix anchor (anchorEnd) | Template has `##%` AND anchorStart=false AND no signPrefix | Numbers not followed by `%` |
| 4 | Enumeration (compact decade) | Range ‚Č§ 50 values | Secondary numbers not matching enumerated values |

## 8. Compiler: 3 Compilation Strategies

- **Enumerated** (‚Č§50 values): single quoted group with compact decade grouping
- **Threshold** (threshold=true): single quoted group with ‚Č•min only ‚ÄĒ no FP from range notation
- **AND Fallback** (>50 values, no threshold): two AND-joined groups

## 10. Number Regex

- `\d{3,}` replaces `[0-9][0-9][0-9]` ‚ÄĒ saves 9 chars per occurrence
- `\d{2,}` replaces `[0-9][0-9][0-9]?` ‚ÄĒ fixes `?` not supported bug
- `\d{4,}` for ‚Č•200, ‚Č•300 etc.
- `[0-9]` still used for single-digit character classes and within decade segments

## 12. Frequent Pitfalls

1. **`!` must be INSIDE quotes with `|`:** `"!A|B"` works, `!"A|B"` does NOT
2. **`.*` does NOT cross block boundaries** ‚ÄĒ use AND (`"X" "Y"`) for cross-block search
3. **`$` end anchor is unreliable** ‚ÄĒ do not use
4. **`?` optional quantifier NOT supported** ‚ÄĒ use `\d{2,}` instead of `[0-9][0-9]?`
5. **Implicit-set bonuses are NOT searchable** ‚ÄĒ filtered out in ETL Step 2b
6. **Core must stay dependency-free** ‚ÄĒ never add npm imports to `src/core/`
7. **Generated JSON is read-only** ‚ÄĒ never edit `public/generated/` by hand
8. **Types in `types.ts` ONLY** ‚ÄĒ no duplicate interface definitions elsewhere
9. **Vendor properties in `vendor-properties.ts` ONLY** ‚ÄĒ canonical source
10. **`reversed=true` for implicit tokens** ‚ÄĒ regex pattern is `"suffix.*(number)%"`
11. **Word truncation works at END only** ‚ÄĒ `"–ļ —Ā–ł"` matches `"–ļ —Ā–ł–Ľ–Ķ"`, mid-word does NOT work. Min 3 significant chars
12. **Item rarity label IS indexed** ‚ÄĒ ¬ę—Ä–Ķ–ī–ļ¬Ľ from ¬ę—Ä–Ķ–ī–ļ–ł–Ļ¬Ľ matches ALL rare items. NEVER use `—Ä–Ķ–ī–ļ–ĺ—Ā—ā` as truncated anchor
13. **`{N,}` quantifier works** ‚ÄĒ `\d{3,}`, `\d{2,}`, `\d{4,}` all valid
14. **Truncation = basic substring** ‚ÄĒ no separate in-game verification needed for unique morphemes
15. **Sign prefix (`\+` and `-`)** ‚ÄĒ provides implicit anchoring against FP from range notation numbers
