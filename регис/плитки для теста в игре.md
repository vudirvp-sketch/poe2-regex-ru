# Плитки для in-game тестирования регекса

> Тест-кейсы для проверки генерируемых regex в PoE2 (RU клиент).
> Каждый тест = предмет + regex для вставки в поиск + ожидаемый результат.
> **Один тест — одна уникальная проверка regex-паттерна. Дубликатов нет.**

---

## Как читать тесты

```
T##  Паттерн: <описание проверяемого паттерна>
Предмет: <название предмета>
Regex: "<regex>"
Ожидание: <подсветить / НЕ подсвечивать>
Проверка: <что именно проверяем — уникально для этого теста>
```

---

## Критические синтаксические правила (справочно)

Эти правила подтверждены в `docs/IN_GAME_TESTS.md`. Здесь — для быстрой ссылки при тестировании.

1. `!` ДОЛЖНО быть внутри кавычек: `"!A|B"` работает, `!"A|B"` — НЕТ
2. `.*` НЕ пересекает границы блоков — каждый мод/implicit/property/state = отдельный блок
3. `.*` направлен — forward only
4. AND через пробел между quoted-группами — order-independent, работает ACROSS blocks
5. `!X` — item-wide, исключает весь предмет если X в ЛЮБОМ блоке
6. Case insensitive

## Группа A: Want + Exclude (`"A" "!B"`)

### A1  Паттерн: want + exclude (exclude не найден)
Предмет: Отвратительное потрясение
Regex: `"к сопротивлению молнии" "!к сопротивлению огню"`
Ожидание: Подсвечивает (молния есть, «к сопротивлению огню» нет на предмете)
Проверка: exclude срабатывает только если текст реально найден

### A2  Паттерн: `!X` item-wide — exclude в другом блоке
Предмет: Отвратительное потрясение
Regex: `"к сопротивлению молнии" "!уклонению"`
Ожидание: НЕ подсвечивает (уклонение в другом блоке → item-wide exclude)
Проверка: `!X` исключает весь предмет если X в ЛЮБОМ блоке, а не только в текущем

### A3  Паттерн: `"want" "!A|B"` — multi-exclude через |
Предмет: Отвратительное потрясение
Regex: `"максимуму здоровья" "!уклонению|маны"`
Ожидание: НЕ подсвечивает (уклонение есть → один из exclude срабатывает)
Проверка: `"!A|B"` исключает если найден A ИЛИ B — `|` работает внутри `!`

---

## Группа B: OR Mode (`"A|B"`)

### B1  Паттерн: `"A|B"` — один из вариантов найден
Предмет: Отвратительное потрясение
Regex: `"к сопротивлению молнии|к сопротивлению огню"`
Ожидание: Подсвечивает (есть молния)
Проверка: `|` внутри кавычек — выбор любого совпадения

### B2  Паттерн: `"A|B" "!C"` — OR + exclude
Предмет: Отвратительное потрясение
Regex: `"к сопротивлению молнии|к сопротивлению огню" "!хаосом"`
Ожидание: Подсвечивает (есть молния, нет хаоса на предмете)
Проверка: OR-match на want + item-wide exclude

### B3  Паттерн: `"A|B"` — ни один не найден
Предмет: Ненавистное потрясение
Regex: `"к сопротивлению молнии|к сопротивлению огню"`
Ожидание: НЕ подсвечивает (нет ни молнии, ни огня)
Проверка: OR не нашёл → предмет отсекается, даже если exclude тоже не сработал бы

---

## Группа C: AND Across Blocks (`"A" "B"`)

### C1  Паттерн: AND — оба мода в разных блоках
Предмет: Отвратительное потрясение
Regex: `"максимуму здоровья" "к сопротивлению молнии"`
Ожидание: Подсвечивает (оба мода есть, в разных блоках)
Проверка: AND — каждый quoted group ищет в своём блоке, order-independent

### C2  Паттерн: AND — implicit + state
Предмет: Призрачный камень
Regex: `"Шанс выпадения" "Осквернено"`
Ожидание: Подсвечивает
Проверка: AND между implicit-блоком и state-блоком

### C3  Паттерн: AND — один не найден
Предмет: Отвратительное потрясение
Regex: `"максимуму здоровья" "к интеллекту"`
Ожидание: НЕ подсвечивает (нет «к интеллекту»)
Проверка: AND требует ОБА условия

## Группа D: `.*` Within Block

### D1  Паттерн: `.*` forward — совпадает
Предмет: Отвратительное потрясение
Regex: `"к сопротивлению.*молнии"`
Ожидание: Подсвечивает («к сопротивлению» стоит ДО «молнии» в одном блоке)
Проверка: `.*` — forward only, совпадение в прямом порядке

### D2  Паттерн: `.*` reverse — НЕ совпадает
Предмет: Отвратительное потрясение
Regex: `"молнии.*к сопротивлению"`
Ожидание: НЕ подсвечивает (обратный порядок в одном блоке)
Проверка: `.*` — forward only, обратный порядок не матчит

### D3  Паттерн: `.*` НЕ пересекает блоки
Предмет: Отвратительное потрясение
Regex: `"максимуму здоровья.*к сопротивлению молнии"`
Ожидание: НЕ подсвечивает (разные блоки)
Проверка: каждый блок — отдельная область поиска, `.*` не пересекает границы

### D4  Паттерн: `.*` пересекает newline внутри многострочного мода
Предмет: Призрачный камень
Regex: `"повышение шанса.*бонусу"`
Ожидание: Подсвечивает (мод «276% повышение шанса критического удара \n +28% к бонусу критического урона»)
Проверка: `.*` работает через перенос строки внутри одного мод-блока

## Группа E: Anchors (`^`, `%`, `: `)

### E1  Паттерн: `^(N)%.*suffix` — ^ anchor (число в начале блока)
Предмет: Потусторонний ордер (39% увеличение количества редких монстров)
Regex: `"^(3[5-9]|[4-9][0-9])%.*редких монстров"`
Ожидание: Подсвечивает (39% ≥ 35%, мод начинается с числа → ^ не мешает)
Проверка: `^` гарантирует что число в начале блока, не в range notation

### E2  Паттерн: `(N)%.*suffix` — % suffix anchor
Предмет: Отвратительное потрясение (+35% к сопротивлению молнии)
Regex: `"(3[1-9]|4[0-9])%.*к сопротивлению молнии"`
Ожидание: Подсвечивает (35% ≥ 31%)
Проверка: `%` после числа отсекает FP от range notation (число без % не матчит)

### E3  Паттерн: `suffix.*: N` — colon anchor (значение ниже порога)
Предмет: Возвышенная Плитка Бездны чемпионов (дополнительных редких монстров: 1)
Regex: `"появляется.*: ([2-9]|[1-9][0-9])"`
Ожидание: НЕ подсвечивает (значение 1 < 2, `: ` гарантирует что 1 после двоеточия не матчит [2-9])
Проверка: `: ` anchor предотвращает FP — число из range notation не попадает после двоеточия

---

## Группа F: Enumerated Range (Decade Grouping)

### F1  Паттерн: `(N1|N2|N3)%.*suffix` — значение в диапазоне
Предмет: Потусторонний ордер (39% редких монстров)
Regex: `"(2[5-9]|3[0-9]|40)%.*редких монстров"`
Ожидание: Подсвечивает (39% в диапазоне 25-40)
Проверка: decade grouping + % anchor работают совместно

### F2  Паттерн: `(N1|N2)%.*suffix` — значение вне диапазона
Предмет: Потусторонний ордер (39% редких монстров)
Regex: `"(2[5-9]|3[0-5])%.*редких монстров"`
Ожидание: НЕ подсвечивает (39% > 35%, верхняя граница не пробита)
Проверка: enumeration НЕ выходит за верхнюю границу — 39 не попадает в 25-35

---

## Группа G: Reversed Regex (Implicit Patterns)

### G1  Паттерн: `suffix.*(N)%` — implicit, текст до числа
Предмет: Призрачный камень (Шанс выпадения путевого камня: +85%)
Regex: `"падения.*([89][0-9]|[1-9][0-9][0-9])%"`
Ожидание: Подсвечивает (85% ≥ 85%)
Проверка: reversed regex — текст.*число% (implicit не dual-indexed)

### G2  Паттерн: `suffix.*(NNN)%` — 3-digit implicit
Предмет: Разрушенный коридор (Шанс выпадения: +110%)
Regex: `"падения.*([1-9][0-9][0-9])%"`
Ожидание: Подсвечивает (110% ≥ 100%)
Проверка: reversed для 3-digit чисел

### G3  Паттерн: `suffix.*(N)` — implicit без % (заряды)
Предмет: Потусторонний ордер (Осталось зарядов - 10)
Regex: `"сь зарядов.*([5-9]|[1-9][0-9])"`
Ожидание: Подсвечивает (10 ≥ 5)
Проверка: implicit reversed без % anchor — заряды без %

---

## Группа H: Dual-Number Prefix (`"От N.*suffix"`)

### H1  Паттерн: `"От N.*suffix"` — prefix anchoring (матчит)
Предмет: Расколотый завиток (Добавляет от 17 до 30 урона от огня к атакам)
Regex: `"От ([1-9][0-9]).*огня к атакам"`
Ожидание: Подсвечивает (17 ≥ 10)
Проверка: prefix «От» привязывает число к первому placeholder в dual-number моде

### H2  Паттерн: `"От N.*suffix"` — мода нет (не матчит)
Предмет: Отвратительное потрясение (нет dual-number мода)
Regex: `"От ([1-9][0-9]).*урона к атакам"`
Ожидание: НЕ подсвечивает
Проверка: предмет не имеет dual-number мода — regex просто не находит совпадение

## Группа I: Специальные символы и форматы

### I1  Паттерн: `[её]` — yofication
Предмет: Расколотый завиток (+32 к силе)
Regex: `"к сил[её]"`
Ожидание: Подсвечивает
Проверка: `[её]` матчит «к силе» — character class с alternation

### I2  Паттерн: `-N` — отрицательное значение
Предмет: Призрачный камень (-11% максимум сопротивлений)
Regex: `"-11"`
Ожидание: Подсвечивает
Проверка: `-` — literal вне `[]`, матчит отрицательные числа

### I3  Паттерн: `N.M` — дробное, `.` = любой символ (включая literal dot)
Предмет: Ненавистное потрясение (Регенерация 15.9 здоровья)
Regex: `"15.9"`
Ожидание: Подсвечивает
Проверка: `.` в PoE2 regex матчит любой символ, включая literal точку

### I4  Паттерн: `N[.]M` — дробное, `[.]` = точная точка
Предмет: Ненавистное потрясение (Регенерация 15.9 здоровья)
Regex: `"15[.]9"`
Ожидание: Подсвечивает
Проверка: `[.]` — exact dot match, более строгий чем `.`

### I5  Паттерн: state text indexed — «Осквернено»
Предмет: Призрачный камень (Осквернено)
Regex: `"Осквернено"`
Ожидание: Подсвечивает
Проверка: state-текст индексируется и доступен для поиска

---

## Группа J: Критический синтаксис — `!` внутри/снаружи кавычек

### J1  Паттерн: `!` снаружи кавычек — НЕ работает
Предмет: любой предмет с «к сопротивлению огню»
Regex: `"к сопротивлению огню" !"уклонению"`
Ожидание: Ничего не подсвечивает (PoE2 не распознаёт `!"..."` как exclude)
Проверка: `!` ДОЛЖНО быть внутри кавычек — `"!A"` работает, `!"A"` — НЕТ
