## Фишка 1: Пороговые паттерны `([X-Y]\d|\d{3,})` вместо enumeration

Риск FP: число из диапазона «(20—30)%» может совпасть с порогом.

### Тест 1A: Порог ≥30% для золота
```
"([3-9]\d|\d{3,})%.*золота"
```
| Предмет | Значение | Ожидание |
|---------|----------|----------|
| #16 Языческий приказ | 32% | ✓ ПОКАЗАТЬ |
| #17 Тревожный суд | 30% | ✓ ПОКАЗАТЬ |
| #10 Космический ордер | 29% | ✗ СКРЫТЬ |
| #6 Древний тайник | 27% | ✗ СКРЫТЬ |
| #3 Потусторонний проект | 25% | ✗ СКРЫТЬ |

### Тест 1B: Порог ≥25% для золота
```
"([2-9]\d|\d{3,})%.*золота"
```
Все 5 (#3, #6, #10, #16, #17) — ПОКАЗАТЬ.

### Тест 1C: ⚠️ FP-ТЕСТ — порог ≥30% + «Бездн»
```
"([3-9]\d|\d{3,})%.*Бездн"
```
ВСЕ 5 предметов Бездны (#7, #9, #11, #25, #26) — СКРЫТЬ.
Если хоть один подсветился — FP от числа в диапазоне «(20—30)%».

### Тест 1D: Порог ≥30% + «путев»
```
"([3-9]\d|\d{3,})%.*путев"
```
6 предметов (#13, #18, #19, #20, #22, #23) — ПОКАЗАТЬ.

### Тест 1E: Порог ≥30% + «редких монстров» — все ниже
```
"([3-9]\d|\d{3,})%.*редких монстров"
```
ВСЕ (#5, #6, #8, #12, #19) — СКРЫТЬ. Если подсветилось — FP от диапазона.

```

```

## Приоритет тестирования

1. **1C** — критический FP-тест (решает: можно ли пороги?)
2. **2A** — дверной тест `{N,}` (решает: фишка 2 вообще работает?)
3. **C1/C2** — `!` + regex-группы (открывает negation-подход)
4. **3A** — `редкост` FP (определяет: можно ли обрезать хвосты?)
5. **1A/1B** — базовые пороги (фундамент)
