## Раунд 1 — Базовая верификация (T1–T5)

### T1 — 2 mandatory + OR
```
"повышение шанса критического удара" "бонусу критического урона монстров" "пробивает|порога состояний"
```
| W1 | W2 | W3 |
|----|----|----|
| ✅ | ❌ | ✅ |

**Вывод:** mixed AND+OR РАБОТАЕТ. OR-группа matчится через любую ALT, отсутствие матчей в OR отбрасывает предмет.

### T2 — 3 mandatory + OR
```
"повышение шанса критического удара" "бонусу критического урона монстров" "Эффективность монстров" "пробивает|порога состояний"
```
| W1 | W2 | W3 |
|----|----|----|
| ✅ | ❌ | ✅ |

**Вывод:** количество mandatory не влияет на механику.

### T3 — Path D (`.*` мост) внутри OR + AND
```
"бонусу критического урона монстров" "Монстры имеют.*порога состояний|Урон монстров.*пробивает"
```
| W1 | W2 | W3 |
|----|----|----|
| ✅ | ❌ | ✅ |

**Вывод:** Path D (same-block AND через `.*` внутри ALT) сохраняется в combined-режиме.

### T4 — `^` на обеих ALT
```
"бонусу критического урона монстров" "^Монстры имеют.*порога состояний|^сопротивлений стихиям"
```
| W1 | W2 | W3 |
|----|----|----|
| ✅ | ❌ | ❌ |

**Вывод:** `^` на второй ALT ломает матч этой ALT. W3 имеет блок «Урон монстров пробивает 13% сопротивлений стихиям», но `^сопротивлений стихиям` требует блок, начинающийся с «сопротивлений стихиям» — не matчит. **`^` только на первой ALT в OR** (KI#45).

### T5 — Лимит 250 chars (rejected)
```
"повышение шанса критического удара" "бонусу критического урона монстров" "Эффективность монстров" "Редкость предметов" "Размер групп монстров" "Шанс выпадения путевого камня" "пробивает|порога состояний|замерзшей земли|бронированы|уклончивы|Путами времени|отравление"
```
Длина: **268 chars** — игра НЕ принимает.

**Вывод:** лимит 250 chars жёсткий в combined-режиме (KI#46). Сплитить на несколько регексов нельзя (AND между регексами ≠ AND внутри одного).

## Раунд 2 — Расширенные сценарии для UI (T6–T10)

### T6 — Две OR-группы + AND между ними
```
"бонусу критического урона монстров" "повышение шанса критического удара|Эффективность монстров" "пробивает|порога состояний"
```
| W1 | W2 | W3 |
|----|----|----|
| ✅ | ❌ | ✅ |

**Вывод:** несколько OPT-групп работают. AND между ними. W2 не matчит — у него нет pierce/status_threshold (вторая OR-группа не matчится).

### T7 — `!` (negation) + AND + OR
```
"!замерзшей земли" "повышение шанса критического удара" "бонусу критического урона монстров" "пробивает|порога состояний"
```
| W1 | W2 | W3 |
|----|----|----|
| ✅ | ❌ | ✅ |

**Вывод:** `!` item-wide работает в combined-режиме. W2 исключён через `!замерзшей земли` (W2 содержит этот аффикс). UI может иметь поле «исключить аффикс».

### T8 — Truncation в combined-режиме
```
"шанса критического удара" "бонусу" "пробивает|порога состояний"
```
| W1 | W2 | W3 |
|----|----|----|
| ✅ | ❌ | ✅ |

**Вывод:** truncation работает в combined-режиме. UI может auto-truncate длинные имена для экономии длины (mitigation для KI#46).

### T9 — OR с пороговыми паттернами + AND
```
"бонусу критического урона монстров" "([6-9]|1[0-6])%.*пробивает|([3-7]\d)%.*порога состояний"
```
| W1 | W2 | W3 |
|----|----|----|
| ✅ | ❌ | ❌ |

**Вывод:** пороги работают в «прямой» форме `N%.*suffix` (W1 matчит через `([3-7]\d)%` = 30-79 → 70%). W3 НЕ matчит из-за **reversed-формы pierce**: блок «Урон монстров **пробивает 13%** сопротивлений стихиям» — «пробивает» ДО числа, а паттерн `([6-9]|1[0-6])%.*пробивает` ожидает число → потом «пробивает». Для reversed нужен паттерн `suffix.*N%` (через `reversed` флаг в AST, уже реализован для MUST в `category-ast-utils.ts`).

### T10 — OR-группа без матчей
```
"повышение шанса критического удара" "бонусу критического урона монстров" "бронированы|уклончивы"
```
| W1 | W2 | W3 |
|----|----|----|
| ❌ | ❌ | ❌ |

**Вывод:** OR-группа без матчей отбрасывает все предметы. Semantics = AND с OR-группой как обязательным элементом (не опциональным).

## Итоговые правила для UI-режима «MIXED» (iter 158)

1. ✅ **Базовый шаблон** `"MUST1" "MUST2" "OPT1|OPT2|OPT3"` — работает (T1–T3).
2. ✅ **Несколько OPT-групп** `"MUST" "OPT1|OPT2" "OPT3|OPT4"` — работают, AND между ними (T6).
3. ✅ **`!` negation** `"!BAD" "MUST" "OPT1|OPT2"` — работает item-wide (T7).
4. ✅ **Truncation** — для экономии длины под лимит 250 (T8, mitigation для KI#46).
5. ✅ **Path D в OPT** `"MUST" "ctx1.*suffix1|ctx2.*suffix2"` — работает (T3).
6. ✅ **Пороги в OPT (прямая форма)** `"MUST" "([X-Y]\d)%.*suffix"` — работает (T9).
7. ⚠️ **Пороги в OPT (reversed)** `"MUST" "suffix.*([X-Y]\d)%"` — нужен `reversed` флаг в AST (T9, не новый KI).
8. ❌ **`^`-anchor** — только на первой ALT в OPT-группе (KI#45, T4).
9. ⚠️ **Лимит 250 chars** — UI счётчик + auto-truncation + ограничения (KI#46, T5).
10. ✅ **OR без матчей** → все предметы скрыты (T10) — UI не должен позволять пустую OPT-группу.
