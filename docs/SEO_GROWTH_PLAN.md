# SEO Growth Plan — единый план роста

> **Дата:** 2026-07-08 (iter 180)
> **Статус:** Активный
> **Контекст:** после iter 175–179 (техническая SEO-база готова) стало понятно, что технической SEO-обёртки недостаточно — сайт физически не виден в выдаче. Два независимых анализа (внешний и внутренний) сошлись на одном диагнозе: проблема не в коде, а во внешних сигналах (ссылки, трафик, ручные шаги в поисковиках).

---

## Постановка проблемы

Техническая часть SEO реализована лучше, чем у большинства инди-инструментов: route-specific title/description/OG/Twitter, canonical, JSON-LD (WebApplication + FAQPage), robots.txt + sitemap.xml на 10 URL, двухслойный пререндеринг (shell + Playwright), IndexNow при деплое, мета-теги верификации Google/Яндекс/Bing. Это **выше базового уровня**.

Однако сайт не появляется в выдаче даже по уникальным фингерпринтам («1 491 аффиксов» и т.п.), которые физически не могут совпасть с другими ресурсами. Причины:

1. **`robots.txt` на project page** — поисковики ищут `robots.txt` строго в корне хоста (`https://vudirvp-sketch.github.io/robots.txt`), а на GitHub Pages project page он лежит по префиксу пути (`/poe2-regex-ru/robots.txt`). Автообнаружение sitemap через `robots.txt` может не срабатывать.
2. **Ручные шаги индексации не завершены** — мета-теги верификации есть в коде, но сама верификация в GSC / Яндекс.Вебмастере / Bing не подтверждена, sitemap не отправлен вручную, URL Indexing API не запрошен.
3. **Ссылочная масса — по факту нулевая.** Ни одной внешней ссылки не найдено. 3 stars / 1 fork / 1 follower на GitHub.
4. **Нет своего домена.** `vudirvp-sketch.github.io/poe2-regex-ru/` — длинный URL, не запоминается, в выдаче рядом с чистыми доменами конкурентов выглядит менее «настоящим», бьёт по CTR.
5. **Яндекс сильнее Google опирается на поведенческие факторы** — у страницы без трафика нет CTR/времени-на-сайте/возвратов → нечем «наградить» на старте. Замкнутый круг.
6. **Маловато текста под длинный хвост запросов** — сайт это тул-экран на категорию, нет раздела готовых примеров regex.
7. **Мелочи** — `<title>` длиннее 60 символов (обрезается в выдаче), `meta keywords` — мёртвый груз (итер 180: исправлено).

---

## Классификация задач

Задачи разделены на 3 buckets по ownership:

- **REPO** — делается в репозитории (этим/следующим iter'ом).
- **MANUAL** — пользователь делает вручную в веб-интерфейсах (GSC, Яндекс.Вебмастер, GitHub Settings, Discord, Steam, DTF).
- **DEFERRED** — отложено пользователем (переезд на домен).

## Bucket 2: MANUAL (пользователь делает вручную)

> Мета-теги верификации уже в коде (`index.html`). Действия ниже — это «сказать поисковику, что ты есть».

### 2.1. Поисковые консоли — ВЫСОКИЙ ПРИОРИТЕТ

- [ ] **Google Search Console.** [search.google.com/search-console](https://search.google.com/search-console) → Добавить ресурс → Префикс URL → `https://vudirvp-sketch.github.io/poe2-regex-ru/` → способ: HTML-тег → «Подтвердить». Мета-тег в `index.html` уже стоит.
- [ ] **Отправить sitemap в GSC.** После подтверждения → Sitemaps → `sitemap.xml` → Submit.
- [ ] **Запросить индексацию** всех 10 URL вручную через URL Inspection → Request Indexing.
- [ ] **Яндекс.Вебмастер.** [webmaster.yandex.ru](https://webmaster.yandex.ru) → Добавить сайт: `https://vudirvp-sketch.github.io/poe2-regex-ru/` (С ПОЛНЫМ ПУТЁМ, не корневой URL) → способ: HTML-тег → «Подтвердить». HTML-файл `yandex_227088c0d89586c7.html` уже в `public/`. Указать регион: Россия.
- [ ] **Bing Webmaster Tools.** [bing.com/webmasters](https://www.bing.com/webmasters) → импорт из GSC (если GSC уже подтверждён) или ручная верификация. Мета-тег `msvalidate.01` уже стоит.

> **Эффект:** в течение недель (не месяцев) — страницы появятся в выдаче по brand-запросам и уникальным фингерпринтам.

### 2.3. Внешние ссылки и дистрибуция — САМЫЙ ВЫСОКИЙ ПРИОРИТЕТ (по эффекту)

> Пользователь сказал, что возьмёт на себя анонсирующие посты на DTF / Steam / форум. Этот раздел — как чеклист.

- [ ] **Steam-гайд** (русский): «Как искать предметы в Path of Exile 2 (Regex и фильтры)» → подробный гайд → в конце ссылка на инструмент. Steam-гайды имеют высочайший приоритет в поиске по игре.
- [ ] **Официальный форум PoE** — раздел «Русскоязычная секция» на `ru.pathofexile.com`. Тема с разбором и ссылкой.
- [ ] **DTF-статья** — «Как я сделал генератор regex для русского PoE2, чтобы не превышать лимит 250 символов». Контент + бэклинк с домена, который уже ранжируется №1 по теме.
- [ ] **VK-паблики по Path of Exile 2** — короткий анонс с гифкой/скриншотом.
- [ ] **Reddit r/PathOfExile2** — пост на английском с пометкой «RU-client tool».
- [ ] **Тематические Discord-серверы** — обычно есть канал «фан-инструменты».
- [ ] **Связь с авторами других тулзов** (Lailloken UI, Exiled-Exchange-2 и т.п.) — Issue на GitHub: «добавьте ссылку на русский генератор рядом с poe2.re».

### 2.4. Поведенческие факторы Яндекса

- [ ] Когда появятся первые посетители — **следить за CTR из выдачи и временем на сайте** в Яндекс.Вебмастере. Если CTR низкий → улучшать title/snippet. Если время на сайте < 30s → проверять, что страница грузится быстро (уже OK — Playwright prerendered).

## Bucket 3: DEFERRED (отложено пользователем)

- [ ] **Свой домен** (`poe2-regex.ru` или подобный) — пользователь явно отложил. Когда вернётся к этому: купить домен → в Settings репо → Pages → Custom domain → добавить CNAME-запись. Хостинг остаётся бесплатным (GitHub Pages). Эффект: буст доверия в Google и Яндексе, короче URL, выше CTR. Также решает проблему с `robots.txt` (будет в корне своего домена).
- [ ] **Любые изменения интерфейса ради SEO** — пользователь явно запретил. Текст на главной можно править (тексты в `<details>` свободны), но layout / chrome / components не трогать.

## Ограничения и контракт с пользователем

1. **Конкурентов для русского клиента нет** (по словам пользователя — проверено). Цель — появляться в выдаче по большинству запросов потенциальных пользователей, а не догонять конкретный сайт.
2. **Переезд на домен — отложен.** Не предлагать, не обсуждать.
3. **Интерфейс не должен пострадать от SEO.** Любые правки, ломающие визуал — запрещены. Тексты на главной (в `<details>`) можно менять свободно.

## Метрики успеха (через 4–8 недель после MANUAL-шагов)

- [ ] В GSC: ≥8 из 10 URL в статусе «Indexed».
- [ ] В Яндекс.Вебмастере: сайт появился в выдаче хотя бы по 5+ запросам (не только brand).
- [ ] По фингерпринту «1 491 аффиксов» (или другой уникальной фразе с сайта) — сайт находится в Яндексе/Google.
- [ ] ≥5 внешних ссылок из разных источников (Steam, DTF, форум, Discord, Reddit/VK).
- [ ] ≥50 уникальных посетителей/неделя (по Яндекс.Метрике или GSC Performance).

Если через 8 недель метрики не двигаются — пересмотреть стратегию (возможно, вернуться к вопросу о домене).
