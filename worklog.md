Task: Анализ бага OR-режима: ложные срабатывания при выборе аффиксов с regexExclude

Work Log:
- 1: Клонирован репозиторий, изучена структура проекта
- 2: Извлечены токены с regexExclude из jewel.json (22 токена), проанализированы паттерны exclude
- 3: Глубоко проанализирован пайплайн: ETL → JSON → AST → Optimizer → Compiler → PoE2 regex
- 4: Выявлена корневая причина: compiler.ts compileInner(AND) всегда оборачивает children в `"..."`, создавая вложенные кавычки внутри OR — PoE2 это не поддерживает
- 5: Симуляция парсинга PoE2 подтвердила: вложенные кавычки ломают OR-структуру, превращая AND-группы с exclude в некорректные AND-группы верхнего уровня
- 6: Проверены все категории: амулеты (82), кольца (52), пояса (11), плиты (8), путевые камни (2) тоже затронуты
- 7: Составлен план исправления: 4 итерации (A: компилятор, B: оптимизатор, C: другие категории, D: in-game верификация)
- 8: Обновлена документация: STATUS.md (итерация 34)

Stage Summary:
- **Баг проанализирован, код не изменён** (только анализ + документация)
- **Корневая причина:** вложенные кавычки при AND-in-OR в compiler.ts
- **Решение:** компиляция AND внутри OR через `()` вместо `"..."` per-child
- **STATUS.md обновлён** — добавлено описание бага, план исправления, затронутые категории
- **Точка остановки:** анализ завершён, исправление не начато
